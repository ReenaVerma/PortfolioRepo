project 4: hit me up

tech: MERN stack - MongoDB, Express.js, ReactJS, Node.js, SASS/SCSS, Bulma, Google APIs, Zomato API, JWT token, FileStack API.

For my final project at General Assembly, I built an app in React, which users can use, to locate other travellers nearby when travelling abroad. The app stores a users location once logged in and other traveller's locations, will then be dropped onto a Google Map on the logged in users Hub Page.

The logged in user can then message visible travellers directly, or search for travellers, using a filter function, (either by username or travel type - eg 'Solo').

Secondary features include timestamps, for when travellers last logged in. As well as seeing travel related content/restaurants nearby, based on geolocation lat/lng.

Built with mobile first in mind.

project 3: crave

tech: MEAN stack - MongoDB, Express.js, AngularJS, Node.js, Google APIs (Places, Servce, Directions), Bulma

Crave App was the result of working on a group project with 2 other classmates. The function of the app is simple - the app is used, when users are too drunk or hungover, to find any nourishment on their way home from a night out!

My role was to develop the Google Maps geolocation function, alternative address search fields, selecting choice of transport, (walking or driving), mapping the origin to destination and allowing users plot results by filter. Filters were displayed as '4 emotional states' - "Hammered" (kebabs), "Hangry (fastfood)", "Hungover (cafes)" and "Hardworking (chinese)". I also developed in step by step directions to be displayed, underneath the plotted route map.

Built with mobile first in mind.

project 2: popup london

tech: Express.js, MongoDB, Node.js, Bcrypt, Flash Messages, Google Maps API, FileStack API

Popup London was built using EJS templating, Node.Js and MongoDB as main technologies. It's a social network, where users can register/log in, to create new popup listings happening around the capital. As logged in users, secondary features include rating and reviewing events, commenting on events and deleting your own events.

Any events added to the database, are then displayed, shown and editable, using REST routing. Users can also filter events based on type, such as 'Art'. And other APIs used included Google Maps and FileStack APIs.
